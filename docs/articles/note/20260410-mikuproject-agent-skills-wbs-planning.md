## はじめに

今回は、生成AI を使って `WBS` 作成を少し幸せにしてみた話です。

`WBS` は大事です。作業を分けて、順番を考えて、日程を置いて、全体を見えるようにする。ちゃんとやると、あとで効いてきます。

でも、正直に言うと、まあまあ面倒です（個人の感想です）。

最初の叩き台を考えること自体は大変ですが、そうではなく、特に面倒なのは、それをデジタルデータとして作ったり、あとから更新したりするところです。

作業を並べ、日付を入れ、表にし、ガントチャートとして見えるようにし、変更があればまた直す。

このあたりのデジタル操作が、地味に手間なのです。

今回、生成AI と `mikuproject` を使ってみたところ、このあたりの面倒がかなり軽くなりました。

※実は `mikuproject` を作ったのは私たちです。

しかも、途中で「実は `Agent Skills` って、こういうところでも効くのか」と、しみじみ感じさせられる場面がありました。

## WBS 作成は、やっぱり面倒

`WBS` を作るとき、最初にしんどいのは白紙から考えるところです。

何を作業として切り出すのか。
どの順番で並べるのか。
どこまで細かく分けるのか。
いつまでに終わらせるのか。

こういうことを考えるのは大事なのですが、最初からきれいに整理しようとすると、けっこう重い作業になりますし、かといって雑に進めると、見落としや混乱のもとになりがちです。

しかも、いったんデジタルデータにしてしまうと、今度はそれを直すのも面倒です。

日程を短くしたい。
作業を分けたい。
順番を変えたい。
別の見せ方にしたい。

そう思っても、表やガントチャートを手で直すとなると、それだけで少し気が重くなります。一方で、表やデジタルデータになっていないと、認識不足や見落としがどうしても起きちゃいます。

## 生成AI に WBS の叩き台を作ってもらう

そこで、生成AI に `WBS` の叩き台を作ってもらいました。

やりたいことや制約を自然文で伝えて、「この前提で `WBS` を作って」とお願いする感じです。

もちろん、いきなり完成品が出てくるわけではありません。

でも、白紙から考えるよりは、かなり楽です。

作業のまとまりや、おおまかな順番や、日程の雰囲気。これらがビジュアルに出てくるだけで、そこから考え始められます。

ゼロから考えるのではなく、出てきた叩き台をビジュアルとともに見ながら考えられます。

「ここは大きすぎるな」
「この順番は違うかも」
「この期間は長すぎるな」
「これちがうやん」

そんなふうに、具体的に見直せるようになります。

この差は、思っていたより大きいです。

## WBS を会話で直せると、だいぶ楽

今回いちばんうれしかったのは、叩き台を作るところだけではありませんでした。

作った叩き台を変更するところも、生成AI と会話して「えいっ」と試せるようになったことです。

`WBS` は、最初に作るのも面倒ですが、直すのも面倒です。

でも、生成AI に「ここをこうしたい」と言うと、まず変更案が出てきます。

たとえば、長すぎる作業を短くしたいとか、作業を分けたいとか、少し日程を詰めたいとか、そういうことを会話で試せます。

もちろん、そのまま信じちゃいけませんよ。

それでも、叩き台を作るのも、叩き台を変更するのも、生成AI と会話して「えいっ」とやってもらえる。

この気軽さは、かなりうれしいものでした。

## 実は `Agent Skills` を使ってました

ここまで、かなり自然に会話しているように見えます。

でも実は、この裏側では `mikuproject` という `Agent Skills` を、生成AIに組み込んで使っていました。

細かい仕組みの話をすると長くなるので、ここでは深く書きません。そこは Qiita 側の記事に分けています。

この記事で言いたいのは、仕組みの説明というより、使ってみた感覚です。

`WBS` の叩き台を作る。
気になるところを直す。
ガントっぽく見る。
Excel や Markdown など、扱いやすい形にする。

そういう流れが、意外なほど会話の中に溶け込んでいました。

## `Agent Skills` すごいね、と思った

今回やってみて、素直に「`Agent Skills` すごいね」と思いました。

というのも、`WBS` 作成の感じが少し変わったからです。

白紙から項目を洗い出して、順番を考えて、日付を入れて、表やガントチャートに整える。そういう作業は大事ですが、正直ちょっと面倒です。

でも、生成AI と会話しながら叩き台を作り、気になるところを直し、見える形にしていくと、作業の重さが少し変わります。

「この条件を入れたらどうなるかな」
「この期間を短くしたらどう変わるかな」
「この作業は分けた方がよさそうだな」

そんなふうに、計画をいじりながら考える感じになります。

`WBS` 作成が楽しい、というのは自分でも少し不思議な感覚です。

でも、今回は本当にそう感じました。

## ただし、責任を取るのは人間

ここで、ひとつ気をつけないといけないこともあります。

生成AI を使って、いい感じの `WBS` が出てきて、さらにそれがガントチャートや Excel のような形になると、見た目としてはかなり「ちゃんとした資料」に見えてきます。

でも、その見た目だけでは、それが本当にちゃんと考えられて作られた計画なのか、まだ叩き台の叩き台くらいのものなのか、判別しにくいことがあります。

ここは少し怖いところです。

見た目が整っていると、つい中身も整っているように感じてしまいます。

作業名が並び、日付が入り、期間が引かれて、ガントチャートになっていると、それだけで計画として成立しているように見えます。

でも実際には、抜けている作業があるかもしれません。順序がおかしいかもしれません。期間が現実的ではないかもしれません。人間が確認すべき工程が足りていないかもしれません。

生成AI や `Agent Skills` は、叩き台を作ったり、形にしたりするところをかなり助けてくれます。

ただし、その `WBS` を承認するかどうか、その計画で進めてよいかどうかの責任まで引き受けてくれるわけではありません。

面倒な叩き台づくりや、形にする作業をかなり楽にしてくれる。でも、最後にその中身を見て、判断して、責任を取るのは人間です。

これからは、見た目がちゃんとしていても、ちゃんと疑う必要がある。

生成AI を使って `WBS` を作ってみて、そのあたりをあらためて感じました。

## まとめ

今回、生成AI と `mikuproject` の `Agent Skills` を使って、`WBS` の叩き台作成と変更を試してみました。

叩き台を作るのも、叩き台を変更するのも、生成AI と会話して「えいっ」とやってもらえる。

この感覚は、思っていた以上に楽でした。

もちろん、生成AI に任せれば `WBS` が完成する、という話ではありません。見た目が整っていても、中身を確認し、承認するかどうかを判断し、責任を取るのは人間です。

それでも、面倒なところを少し軽くしてくれるだけで、`WBS` を考える作業の感じはかなり変わります。

さて、こんどは何の `WBS` を作ってみようかな。楽しくなってきました。
